The primary DB table that this IDataSource reads and writes to and from. There

might be more than one source buffer for the ABL DATA-SOURCE, but this one is this IDataSource object's reason for existence (basically).

The standard datasource populates buffers; this property makes it easier to access those

buffers.

TenantFilterFieldName

CHARACTER PROTECTED GET SET
The default field name in the schmema used to apply a tenant filter. By default,

this is used with the Primary table; subsidiary tables should join off the primary table's values, although there's no requirement for that (ie they could also build their own tenant filters, in slightly different ways if needed). The defaults should suffice for most cases. I think.

FetchData

OVERRIDE PUBLIC VOID FetchData()
Perform fetch: populate the previously-passed buffer from the

physical data store. The caller needs to call GetData() afterwards if they want any additional information about the fetch call (stuff that's in ITableContext). Note: - this method will not be called when Datasets are involved, since the FILL() method will probably be called instead. The code below is for illustrative purposes. - there are no events (before-fill etc) fired since this code doesn't use prodataset; we should therefor create our own events and fire them if we so desire.
