FetchData

OVERRIDE PUBLIC VOID FetchData()
Perform fetch: populate the previously-passed buffer from the

physical data store. The caller needs to call GetData() afterwards if they want any additional information about the fetch call (stuff that's in ITableContext). Note: - this method will not be called when Datasets are involved, since the FILL() method will probably be called instead. THe code below is for illustrative purposes. - there are no events (before-fill etc) fired since this code doesn't use prodataset; we should therefor create our own events and fire them if we so desire.

Returns:
ITableContext - the context for the request (num records, position etc).

GetTableHandle

HANDLE GetTableHandle(CHARACTER pcTableName)
IQuery:ITableOwner implementation. The table name passed in here will be the

name of the database table (say) that this data source uses to populate the table represented by the HandleDataTarget property.
Parameters:
character - A table or buffer name. The implementer willknow how to interpret the name. The name passed in is the name that the query will use together with its IQueryDefinition.
Returns:
A buffer handle corresponding to the requested name.
