AttachDataStoreToTarget

VOID AttachDataStoreToTarget()
Attaches the actual datasource - this could be an ABL datasource, or some other

mechanism (XML or a service or something else). These are specific to each datasource instance, and thus is abstract. For example, for an ABL/ProDataSet data source: DATA-SOURCE srcCar:HANDLE:prefer-dataset = TRUE. hBuffer:ATTACH-DATA-SOURCE(DATA-SOURCE srcCar:HANDLE,"").

SetDataTarget

PUBLIC VOID SetDataTarget(HANDLE phData)
This SetDataTarget override is provided since the bulk of datatargets in ABL

will be buffers (possibly datasets). An exception will be thrown if the handle has a type of anything other than 'buffer' or 'dataset'.
Parameters:
handle - A buffer or dataset handle that is the data target.

SetDataTarget

PUBLIC VOID SetDataTarget(LONGCHAR pcData, DataFormatEnum poDataTargetFormat, DataTypeEnum poDataTargetType)
SetDataTarget() methods are called by the DataAccess object, to specify

the target for the DataSource's operations (usually the fill/fetch). This data is formatted per the DataTargetFormat above; and is stored in the message as either LONGCHAR, MEMPTR, HANDLE or P.L.O.
